### `appointment`
| Column | Type | Constraints | Notes |
|--------|------|-------------|-------|
| `id` | SERIAL | PRIMARY KEY | — |
| `arrival_id` | VARCHAR(50) | INDEX | PIN; auto-generated by trigger (BR-20) |
| `booking_reference` | VARCHAR(50) | NOT NULL, FK → booking.reference | (BR-08) |
| `driver_license` | VARCHAR(50) | NOT NULL, FK → driver.drivers_license | (BR-08) |
| `truck_license_plate` | VARCHAR(20) | NOT NULL, FK → truck.license_plate | (BR-08) |
| `terminal_id` | INTEGER | NOT NULL, FK → terminal.id | (BR-08) |
| `gate_in_id` | INTEGER | FK → gate.id | Entry gate (BR-09) |
| `gate_out_id` | INTEGER | FK → gate.id | Exit gate; nullable (BR-09) |
| `scheduled_start_time` | TIMESTAMP | — | — |
| `expected_duration` | INTEGER | — | Minutes |
| `status` | ENUM(appointment_status) | DEFAULT 'scheduled' | Primary flow states written by app: `scheduled → in_transit → in_process → completed \| canceled`. Values `unloading` and `delayed` remain in the PG enum for backward compat but are **never written** by application code — they are computed sub-states. (BR-15) |
| `version` | INTEGER | NOT NULL, DEFAULT 1 | Optimistic concurrency (BR-48, Guardrail 6) |
| `notes` | TEXT | — | — |
| `highway_infraction` | BOOLEAN | DEFAULT FALSE | (BR-42) |

**Triggers**: `trg_generate_arrival_id` (arrival_id via sequence), `trg_check_visit_completion` (auto-complete on visit out_time).

**ORM computed properties** (never stored in DB — derived at read time by `sql_models.Appointment`):

| Property | Type | Derivation |
|----------|------|------------|
| `is_delayed` | `bool` | `status == 'in_transit'` AND `now() > scheduled_start_time + DELAY_TOLERANCE_MINUTES` |
| `is_unloading` | `bool` | `status == 'in_process'` AND active `Visit` exists with `Visit.state == 'unloading'` AND `Visit.out_time IS NULL` |
| `computed_status` | `str` | Primary status → sub-state override: `in_transit + is_delayed → 'delayed'`; `in_process + is_unloading → 'unloading'`; otherwise = raw `status` |

**API response fields** (populated by `AppointmentResponse` Pydantic model):

| Field | Source | Notes |
|-------|--------|-------|
| `status` | `computed_status` | Backward-compat field for existing frontend consumers |
| `display_status` | `computed_status` | Explicit alias for new consumers; always equals `status` |
| `primary_status` | raw `Appointment.status` | Actual DB state; never `'delayed'`/`'unloading'` |
| `is_delayed` | `Appointment.is_delayed` | Boolean sub-state flag |
| `is_unloading` | `Appointment.is_unloading` | Boolean sub-state flag |
| `highway_infraction` | `Appointment.highway_infraction` | Pre-existing boolean orthogonal flag |

## MongoDB

### `agent_detections`
Document shape (variable):
```json
{
  "detection_id": "det_gate1_...",
  "truck_id": "TRUCK-...",
  "agent_type": "AgentA|AgentB|AgentC",
  "gate_id": 1,
  "timestamp": "ISODate",
  "detection_data": { "license_plate": "AB-12-CD", "confidence": 0.92, ... },
  "processing": { "consumed_by_decision_engine": false, ... }
}
```
**Indexes**: `idx_detection_id` (UNIQUE), `idx_truck_timestamp`, `idx_agent_gate_timestamp`, `idx_license_plate` (sparse), `idx_agent_detections_ttl` (TTL 30d).
**Writers**: outbox worker projection only (DW-03 resolved Phase 7).

### `decision_events`
Document shape:
```json
{
  "decision_id": "dec_gate1_...",
  "truck_id": "TRUCK-...",
  "gate_id": 1,
  "appointment_id": 42,
  "agent_detections": { "truck_detection": {}, "license_plate_detection": {}, "hazmat_detection": {} },
  "decision_engine": { "decision": "ACCEPTED", "decision_reason": "...", ... },
  "operator_decision": null,
  "final_decision": "ACCEPTED",
  "final_decision_source": "agent|operator",
  "postgres_updates": { "appointment_updated": true, "new_status": "in_process", ... },
  "timing": { ... },
  "created_at": "ISODate",
  "version": 1
}
```
**Indexes**: `idx_decision_id` (UNIQUE, partial filter `{decision_id: {$type: "string"}}`), `idx_truck_created` (compound, non-unique — BR-37 spec gap), `idx_gate_decision_created`, `idx_manual_review` (sparse), `idx_pipeline_performance`, `idx_decision_license_plate` (sparse), `idx_decision_events_ttl` (TTL 90d).
**Writers**: outbox worker projection only (DW-03, PD-02 resolved Phase 7). `persist_decision_event` retained with `DeprecationWarning`.

### `notifications`

```json
{
  "gate_id": 1,

  "message": "...",
  "type": "success|info|warning|error",
  "appointment_id": 42,
  "license_plate": "AB-12-CD",

  "created_at": "ISODate"
```

**Writers**: `cmd_send_notification` via UoW + outbox projection (DW-01 resolved Phase 5). TTL 30d (`idx_notif_ttl`).

### `statistics_hourly`
Document shape:
```json
{
  "gate_id": 1,
  "hour_bucket": "ISODate",
  "detections": { "total": 12, "by_agent": {}, "avg_confidence": {} },
  "decisions": { "total": 10, "by_type": {} },
  "computed_at": "ISODate",
  "version": 1
}
```
**Indexes**: `idx_gate_hour_unique` (UNIQUE, `gate_id` + `hour_bucket`), `idx_stats_hourly_ttl` (TTL 400d).
**Writers**: `scripts/statistics_aggregator.py` scheduled job (DW-07 resolved Phase 5).

### `statistics_daily`
**Indexes**: `idx_stats_daily_gate_day` (UNIQUE, `gate_id` + `day_bucket`), `idx_stats_daily_ttl` (TTL 400d).

### `operator_performance`
**Indexes**: `idx_op_perf_operator_time` (`operator_id` + `computed_at` DESC), `idx_op_perf_ttl` (TTL 400d).

### `company_metrics`
**Indexes**: `idx_company_metrics_nif_time` (`company_nif` + `computed_at` DESC), `idx_company_metrics_ttl` (TTL 400d).

### `appointments_read` (CQRS read model)
**Indexes**: `idx_appts_read_appointment_id` (UNIQUE), `idx_appts_read_driver_status` (`driver_license` + `status`).
**Status**: index created (Phase 9); outbox projection to populate collection is pending (no `AppointmentCreated` event emitted yet — BR-38 partial).

### `alerts_read` (CQRS read model)
Projection target for `AlertRaised` / `AlertResolved` outbox events (Phase 6).

## Redis Key Patterns

| Key Pattern | TTL | Operation | BR | Notes |
|-------------|-----|-----------|-----|-------|
| `dedup:event:{event_id}` | 300s | SET NX | BR-30 | Event-id dedup gate |
| `plate:{plate}:gate:{id}:tb:{bucket}` | 300s | SET NX | BR-31 | Time-window dedup for LP+gate |
| `decision:plate:{plate}:gate:{id}:tb:{bucket}` | 3600s | SETEX (JSON) | BR-32 | Decision cache |
| `appointment:{id}:details` | 1800s | SETEX (JSON) | BR-33 | Hot appointment cache |
| `lp_lookup:{plate}:appointments` | 600s | SETEX (JSON) | BR-34 | LP → appointment list |
| `counter:gate:{id}:hour:{ts}:{metric}` | 7200s | INCRBY | BR-35 | Real-time gate counters |
| `pending_review:{event_id}` | 1800s | SETEX (JSON) | BR-36 | MANUAL_REVIEW correlation; PG-backed (`pending_reviews` table); key is event_id since Phase 4 |
| `ratelimit:api:{endpoint}:{client}` | 60s | INCR | — | Rate limiting window (`redis.check_rate_limit`) |
| `session:driver:{license}` | 3600s | SETEX (JSON) | — | Driver session (moved from PG in Phase 2) |
| `session:operator:{num_worker}` | 3600s | SETEX (JSON) | — | Operator session |
| `session:manager:{num_worker}` | 3600s | SETEX (JSON) | — | Manager session |
| `stats:gate:{id}:{date}` | 30s | SETEX (JSON) | — | Stats endpoint cache |
